Tankerz | Unity | Personal

  • Context: Personal Project, solo

Presentation

The second game I created for the arcade machine, Tankerz is a game playable by 1 to 2 players locally. Players each control a tank that must shoot all enemies and score as many points as possible before dying. They can raise and lower their cannon and move with their joystick. The B1 button allows them to shoot.

The enemies are quite varied: drones, helicopters, soldiers (two types) and explosive barrels.

What This Game Taught Me

Each level is randomly generated from prefabricated terrain parts, and the same goes for enemy spawning. Thus, each game is different, and this mechanic taught me how to implement this type of procedural generation, a simple but essential skill in game dev!

But THE main skill this game taught me was inheritance in programming. In other words, we can create a base script that shares its code with child scripts. This not only optimizes code lines by avoiding redundancies, but also allows us to manipulate different types of objects generically.

For example, all destructible elements in the game have their own script (like Helicopter.cs or Soldier.cs), but these scripts derive from a parent script named Destructible.cs. This script contains common properties like health points, explosion prefabs, etc. Thus, each game element can have a health bar and a specific type of destruction while maintaining its unique characteristics (for example, the helicopter flies and the soldier shoots).

This inheritance principle will be heavily reused in my game Dashing Krew, which I'll present in another post!

Working with Others

Since Tankerz is designed for an arcade machine, we had to implement a highscore system and specific inputs for the machine's buttons. Unlike classic controls (keyboard or analog joystick), arcade machines use special buttons. For this, our teacher, Jules, created a package called Anatidae, which handles these inputs and the highscore system. This package, available on GitHub, had to be read, understood, and integrated into my project to get the final game.

The biggest difficulty was related to the highscore system: on the arcade machine, scores are stored on the school's internet network. However, during development, I had to configure a mini-local server on my computer, a task that Jordan Septier from 8 months ago had no idea how to do. But in the end, I managed!
