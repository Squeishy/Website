Présentation

Suite à la création de mon entreprise L’usine Canard, je me suis dit qu’il serait super chouette d’avoir un site pour présenter mes jeux et mes coordonnées. En seulement 20 jours, le site lusinecanard.fr a vu le jour !

Les pages

Le site web contient 6 pages principales :

  • Index : Un hub qui mène à toutes les autres pages.

  • Mes créations : Une page qui énumère toutes mes créations (Tankerz, Ducky Tales, Spiral Maker, etc.).

  • À venir : Une page qui tease les projets en cours (au moment où j’écris, c’est Dashing Krew).

  • À propos de moi : Une présentation personnelle.

  • Me contacter : Une page contenant mes liens vers mes réseaux sociaux.

  • Dossier de presse : Une page regroupant tous les assets que j’utilise ou que j’ai créés.

Les langues et le SEO, référencement naturel

Pour améliorer l’expérience utilisateur et booster le SEO, j’ai ajouté des traductions en français et en anglais sur toutes les pages. Deux options s’offraient à moi :

  • Je pouvais juste utiliser un outil externe comme Google Traduction. C’est rapide, mais les résultats peuvent être imprécis.
  • Ou je pouvais utiliser des fichiers JSON pour stocker toutes les traductions possibles. Cette méthode prend plus de temps, mais elle offre deux gros avantages : Premièrement, les traductions sont personnalisées et précises, car vous les gérez vous-même. Et deuxièmement, cela améliore le référencement du site.

J’ai donc choisi la méthode JSON, mais cela nécessitait plusieurs étapes :

J’ai bien évidement choisir la technique des fichiers JSON, mais il m'a fallu quelques éléments : Tout d'abord, il a fallu donner une id unique pour chaque élément à traduire de mon site web, c'est-à-dire les balises p, a, h1 à h6, les span, mais aussi les imgages ! Une fois fait, vous vous rendrez sans doute compte que certaines balises ne fonctionnent pas de la même manière. Par exemple, le texte d'une balise <p> se trouve entre les deux balisent <p></p>, mais si on prend un lien à présent, cela devient ça par exemple :

<a href="index.html" title="Go to the home page">Home Page</a>

Ici, nous avons du texte entre les balises <a></a>, mais aussi dans l'attribut title="", ce qui veut dire qu'il va falloir modifier cela ! Même chose pour les images qui utilisent des alts !

Mais du coup, comment fait-on pour différencier un texte qui se trouve entre deux balises, un texte qui se trouve dans un alt et un texte qui se trouve dans un title ? Eh bien avec notre structure JSON, voici un exemple de ce que j'ai fait pour mon site web :

La mise en ligne

Une fois mon site web finit, j'ai du acheter un nom de domaine (lusinecanard.fr) puis j'ai uploader mon site web sur mon VPS pulseheberg (hébergeur) avec FileZilla, que je métrise ! 

Dernière étape, il fallait configurer les fichier dans sites-available d'Apache2, ce qui s'est fait à coup de CMD (l'invite de commande) afin de pouvoir rediriger l'addresse lusinecanard.fr à l'addresse ip 193.168.146.201. (mon site web)

Une fois ça fait, il fallait relancer Apache2, et c'était tout bon !
